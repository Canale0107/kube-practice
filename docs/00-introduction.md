# イントロダクション

## Kubernetesとは何か

**Kubernetes**（クーバネティス、略してK8s）は、コンテナ化されたアプリケーションを自動的にデプロイ、スケール、管理するためのオープンソースのプラットフォームです。

### 簡単に言うと

Kubernetesは、**コンテナを動かすための「オーケストレーション（編成）ツール」**です。

- コンテナをどこで動かすか決める
- コンテナが壊れたら自動で復旧する
- 負荷が増えたら自動で増やす
- 複数のサーバーを1つの大きなコンピュータのように扱う

### なぜ「Kubernetes」という名前？

Kubernetesはギリシャ語で「操舵手（パイロット）」を意味します。船の舵を取るように、コンテナの運用を管理することから名付けられました。

## なぜKubernetesを学ぶのか

### 1. 業界標準

Kubernetesは、コンテナオーケストレーションの**デファクトスタンダード**（事実上の標準）です。

- Google、Amazon、Microsoftなどの大手クラウドプロバイダーがサポート
- 多くの企業が本番環境で使用
- エンジニアの求人でも頻繁に要求されるスキル

### 2. スケーラビリティ

- アプリケーションの負荷に応じて自動的にスケール（拡張・縮小）
- 数台から数千台のサーバーまで管理可能

### 3. 高可用性（High Availability）

- サーバーやコンテナが壊れても自動で復旧
- ダウンタイムを最小限に抑える

### 4. ポータビリティ

- 同じ設定で、ローカル、クラウド、オンプレミスなど、どこでも動かせる
- ベンダーロックインを避けられる

### 5. エコシステム

- 豊富なツールやプラグイン
- 活発なコミュニティとサポート

## Kubernetesが解決する問題

### 問題1: 手動管理の限界

**従来の方法:**
```
サーバーにアプリをインストール
→ 設定ファイルを編集
→ サービスを起動
→ 問題が起きたら手動で対応
```

**Kubernetes:**
```
設定ファイルを書く
→ Kubernetesが自動で管理
→ 問題が起きても自動で復旧
```

### 問題2: スケーリングの難しさ

**従来の方法:**
- 負荷が増えたら手動でサーバーを追加
- 設定を各サーバーに手動で反映
- 負荷が減っても手動で削除

**Kubernetes:**
- 負荷に応じて自動でスケール
- 設定は一度書けば自動で反映
- 不要になったら自動で削除

### 問題3: 環境の違い

**従来の方法:**
- 開発環境と本番環境で設定が違う
- 「開発では動いたのに本番で動かない」

**Kubernetes:**
- 同じ設定ファイルでどの環境でも動く
- 環境ごとの違いは設定で管理

## Kubernetesの全体像

```
┌─────────────────────────────────────┐
│      Kubernetes Cluster             │
│                                     │
│  ┌──────────┐    ┌──────────┐      │
│  │  Master  │    │  Node    │      │
│  │  Node    │───▶│  (Worker)│      │
│  │          │    │          │      │
│  │  - API   │    │  - Pods  │      │
│  │  - etcd  │    │  - kubelet│     │
│  │  - Sched │    │          │      │
│  └──────────┘    └──────────┘      │
│                                     │
└─────────────────────────────────────┘
         ▲
         │
    kubectl (あなたのコマンド)
```

### 主要なコンポーネント

1. **Master Node（マスターノード）**
   - クラスタ全体を管理
   - APIサーバー、スケジューラー、etcdなど

2. **Worker Node（ワーカーノード）**
   - 実際にPod（コンテナ）を実行
   - kubelet、kube-proxyなど

3. **kubectl**
   - Kubernetesを操作するコマンドラインツール
   - あなたが使う主要なツール

> **このプロジェクトでは**: kindが、これらのコンポーネントをすべてDockerコンテナ内に作成します。本番環境と同じ構造を、ローカルで安全に体験できます。

## 学習の心構え

### 1. 完璧を目指さない

- Kubernetesは複雑なシステムです
- 最初からすべてを理解する必要はありません
- まずは動かしてみることが大切

### 2. エラーは学習の機会

- エラーは「何が間違っているか」を教えてくれます
- エラーメッセージを読む習慣をつけましょう
- トラブルシューティングは重要なスキルです

### 3. 手を動かす

- 読むだけでは理解できません
- 実際にコマンドを実行してみましょう
- 変更を加えて、何が起こるか観察しましょう

### 4. 段階的に学ぶ

- 一度にすべてを学ぼうとしない
- 基本から始めて、少しずつ理解を深める
- このドキュメントは順を追って読むことを推奨します
