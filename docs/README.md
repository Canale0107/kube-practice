## 🎯 学習の進め方

### 完全初心者の場合

1. **イントロダクション**を読んで、Kubernetesの全体像を把握
2. **前提知識**でコンテナとDockerの基礎を理解
3. **セットアップガイド**に従って環境を構築
4. **Kubernetes基本概念**で用語と概念を学習
5. **ハンズオンガイド**に従って実際に操作
6. 問題が起きたら**トラブルシューティング**を参照
7. さらに学びたい場合は**次のステップ**へ

### コンテナ経験がある場合

1. **イントロダクション**を軽く読む
2. **セットアップガイド**で環境構築
3. **Kubernetes基本概念**で用語を確認
4. **ハンズオンガイド**で実践
5. 必要に応じて他のドキュメントを参照

## 💡 このドキュメントの特徴

- **段階的学習**: 基礎から応用まで順を追って学習できる
- **実践的**: 実際に手を動かしながら学べる
- **初心者向け**: 専門用語は丁寧に説明
- **具体例**: 実際のコマンドと出力例を豊富に掲載
- **トラブル対応**: よくある問題と解決方法を網羅

## 📖 用語集

ドキュメント内で使用される主要な用語：

- **Pod**: Kubernetesでコンテナを実行する最小単位
- **Deployment**: Podの設計図。複数のPodを管理
- **Service**: Podへのアクセスを提供するエンドポイント
- **ConfigMap**: 設定データを保存するリソース
- **kubectl**: Kubernetesを操作するコマンドラインツール
- **kind**: ローカルでKubernetesクラスタを作成するツール
